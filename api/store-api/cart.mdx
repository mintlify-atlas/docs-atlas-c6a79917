# Cart API

The Cart API provides endpoints for managing the shopping cart for the current session or logged-in user. All cart operations return the complete cart state, making it easy to keep your frontend synchronized with the server.

## Overview

The Cart API allows you to:

- Retrieve the current cart state
- Add and remove items
- Update item quantities
- Apply and remove coupons
- Update customer billing and shipping addresses
- Select shipping methods
- Calculate taxes and totals

<Note>
All POST endpoints require either a [Nonce Token](#authentication) or a [Cart Token](#cart-tokens) for security. GET requests to `/cart` do not require authentication.
</Note>

## Base Endpoint

All cart endpoints use the base path:

```
/wp-json/wc/store/v1/cart
```

## Authentication

### Nonce Tokens

For traditional WordPress applications, use nonce tokens for write operations:

```bash
curl --header "Nonce: 12345" \
  --request POST \
  https://example-store.com/wp-json/wc/store/v1/cart/add-item
```

Generate nonces using WordPress:

```php
$nonce = wp_create_nonce( 'wc_store_api' );
```

<Warning>
After each request, an updated `Nonce` header is returned in the response. You must use this updated nonce for subsequent requests or they will fail.
</Warning>

### Cart Tokens

For headless applications, use Cart Tokens instead of nonces:

```bash
curl --header "Cart-Token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  --request GET \
  https://example-store.com/wp-json/wc/store/v1/cart
```

Cart Tokens are JWT-based tokens returned in the `Cart-Token` response header when you make any request to a cart endpoint. Store this token and include it in subsequent requests.

**How to obtain a Cart Token:**

1. Make a GET request to `/wc/store/v1/cart`
2. Extract the `Cart-Token` from response headers
3. Use this token for all future cart and checkout operations

```javascript
const response = await fetch('https://example-store.com/wp-json/wc/store/v1/cart');
const cartToken = response.headers.get('Cart-Token');
localStorage.setItem('cartToken', cartToken);
```

<Note>
When using Cart Tokens, nonce tokens are not required. This makes them ideal for headless implementations where cookies are not available.
</Note>

## Add Item to Cart

Add a product or variation to the cart.

```http
POST /wp-json/wc/store/v1/cart/add-item
```

### Parameters

<ParamField path="id" type="integer" required>
  The product or variation ID to add to cart
</ParamField>

<ParamField path="quantity" type="integer" required>
  Quantity to add (must be at least 1)
</ParamField>

<ParamField path="variation" type="array">
  For variable products, an array of attribute objects with `attribute` and `value` keys
</ParamField>

### Request

<CodeGroup>
```bash Simple Product
curl --header "Nonce: 12345" \
  --request POST \
  "https://example-store.com/wp-json/wc/store/v1/cart/add-item?id=100&quantity=2"
```

```bash Variable Product
curl --header "Nonce: 12345" \
  --request POST \
  --header "Content-Type: application/json" \
  --data '{
    "id": 150,
    "quantity": 1,
    "variation": [
      {
        "attribute": "pa_color",
        "value": "blue"
      },
      {
        "attribute": "pa_size",
        "value": "large"
      }
    ]
  }' \
  "https://example-store.com/wp-json/wc/store/v1/cart/add-item"
```

```javascript Fetch API
const response = await fetch('/wp-json/wc/store/v1/cart/add-item', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Nonce': nonce
  },
  body: JSON.stringify({
    id: 100,
    quantity: 1,
    variation: [
      { attribute: 'pa_color', value: 'red' },
      { attribute: 'pa_size', value: 'medium' }
    ]
  })
});

const cart = await response.json();
const updatedNonce = response.headers.get('Nonce');
```
</CodeGroup>

### Variation Attribute Naming

When adding variations, attribute naming is important:

**Global Attributes** use slugs with `pa_` prefix:
- Attribute "Color" → `pa_color`
- Attribute "Size" → `pa_size`

**Product-Specific Attributes** can use either:
- The attribute name (case-sensitive): `Size`, `Autograph ✏️`
- The attribute slug: `attribute_size`, `attribute_autograph-%e2%9c%8f%ef%b8%8f`

### Response

Returns the full cart object (same as [Get Cart](#get-cart)) on success.

<CodeGroup>
```json 201 Created
{
  "items": [
    {
      "key": "c74d97b01eae257e44aa9d5bade97baf",
      "id": 100,
      "quantity": 2,
      "name": "Premium T-Shirt",
      ...
    }
  ],
  "totals": { ... },
  ...
}
```

```json 400 Error - Invalid Product
{
  "code": "woocommerce_rest_cart_invalid_product",
  "message": "This product cannot be added to the cart.",
  "data": {
    "status": 400
  }
}
```
</CodeGroup>

### Adding Multiple Items (Batch)

To add multiple items in one request, use the batch endpoint:

```bash

```

```json

      "method": "POST",
      "body": { "id": 27, "quantity": 2 },
      "headers": { "Nonce": "1db1d13784" }
    }
  ]
}
```

## Error Responses

### Standard Error

```json
{
  "code": "woocommerce_rest_cart_invalid_product",
  "message": "This product cannot be added to the cart.",
  "data": {
    "status": 400
  }
}
```

### Conflict Error (409)

When cart state changes during a request, a 409 response includes the current cart:

```json
{
  "code": "woocommerce_rest_cart_invalid_key",
  "message": "Cart item no longer exists or is invalid.",
  "data": {
    "status": 409,
    "cart": {
      "items": [],
      "coupons": [],
      "totals": { ... },
      ...
    }
  }
}
```

This allows your application to synchronize with the server without an additional request.

## Common Error Codes

| Code | Description |
|------|-------------|
| `woocommerce_rest_cart_invalid_product` | Product cannot be added (out of stock, invalid, etc.) |
| `woocommerce_rest_cart_invalid_key` | Cart item key doesn't exist or is invalid |
| `woocommerce_rest_cart_coupon_error` | Coupon is invalid or cannot be applied |
| `woocommerce_rest_invalid_nonce` | Nonce token is invalid or expired |
| `woocommerce_rest_missing_nonce` | Nonce token is required but not provided |

## Understanding Price Fields

Prices in the Store API are returned as **strings representing the smallest currency unit** (e.g., cents for USD):

```json
{
  "prices": {
    "price": "1800",           // $18.00
    "regular_price": "2000",   // $20.00
    "sale_price": "1800",      // $18.00
    "currency_code": "USD",
    "currency_symbol": "$",
    "currency_minor_unit": 2,  // 2 decimal places
    "currency_prefix": "$",
    "currency_suffix": ""
  }
}
```

To display the price correctly:

```javascript
const priceValue = parseInt(cart.items[0].prices.price);
const minorUnit = cart.items[0].prices.currency_minor_unit;
const displayPrice = (priceValue / Math.pow(10, minorUnit)).toFixed(minorUnit);
// displayPrice = "18.00"
```

## Cart Item Keys

Each cart item has a unique `key` (MD5 hash) based on:
- Product ID
- Variation ID (if applicable)
- Variation attributes
- Custom cart item data

The same product with different attributes will have different keys, allowing multiple variations of the same product in the cart.

## Extensions

The `extensions` object in responses can contain additional data added by plugins:

```json
{
  "items": [ ... ],
  "extensions": {
    "my-plugin": {
      "custom_data": "value"
    }
  }
}
```

Plugins can extend cart data using the WooCommerce Store API extensibility system.
